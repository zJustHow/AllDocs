# 操作手册编写规范

面向**撰写并上传至 AllDocs** 的操作指南、维修手册编写人员。系统架构与入库配置见 [README](../README.md)。

**核心原则：** 可检索的主体是**文字**；插图/表格通过**图号**与正文步骤配对，问答时展示在对应步骤下方，而非堆在文末。

---

## 1. 推荐格式

**正式手册请用带完整书签、可选中文本层的 PDF**（非纯扫描件）。PDF 同时提供章节结构、页码、版面坐标与内嵌图/表，检索、引用跳转与插图对齐效果最好。


| 场景     | 格式                         |
| ------ | -------------------------- |
| 正式手册   | **PDF**（首选）                |
| 内部草稿   | Word → 定稿后导出 PDF 上传        |
| 技术文档   | Markdown → 含大量插图时仍建议输出 PDF |
| 短文、无结构 | TXT / HTML（不适合长篇手册）        |


Word 用内置「标题 1–6」样式；Markdown 用 `#` 标题。单独上传的图片仅整图 OCR，无章节结构。

当前各格式的实际处理边界：

| 格式 | 入库结构 | 前端预览 | 重要限制 |
|------|----------|----------|----------|
| PDF | 书签、页码、版面坐标、矢量表与内嵌图 | 按页渲染 | 首选；扫描页可能需要 OCR |
| DOCX | 标题样式与段落文本 | 服务端转换的安全 HTML | 预览可显示表格，但表格单元格和内嵌图片当前不进入检索索引 |
| Markdown | `#` 标题与正文 | 文本预览 | 不提取插图 asset |
| TXT / HTML | 单段正文，无 section | 文本 / 受限 HTML 预览 | HTML 标签会在入库时移除，不保留页面结构 |
| PNG / JPG / WEBP | 整图 OCR | 图片预览 | 无章节结构 |

因此，Word 适合撰写源稿，但凡表格、插图、页码或引用跳转属于关键内容，都应先导出 PDF 再上传。不要把“预览中看得到”理解为“检索索引中已经包含”。

## 2. 内容与排版

- **文字优先**：步骤、参数、报警码、警告等写成正文或文字表，不要用图片替代。
- **报警码与型号**：报警码（如 `E001`）、零件号、型号须以**可复制、可检索的正文**出现；Agent 会通过 `search_keyword` 做 BM25 精确匹配，仅印在图上或页眉里很难命中。
- **图片作补充**：外观示意、接线图、界面截图等看图更直观的内容；一图一注，不重复粘贴相同大图。
- **表格**：在源稿中使用可编辑的 Word 表格，并在正式上传版中导出为 **PDF 矢量表**；不要用截图、扫描图或整页图片充当表格。DOCX 预览虽能显示表格，当前直接入库并不索引其单元格。单元格内不要嵌图片。表格做成图片时，管理员可在 **设置页**（`/settings`）或 `.env` 中开启 `INGEST_CAPTION_ENABLED` 尝试 VLM 识别并 OCR 为结构化表，但效果远不如矢量表。**尽量不要拆分表格**；若因版面必须跨页拆分，各段应视为同一表，**每段都加相同表号**（见 §3）。
- **步骤**：用编号列表书写，关键信息不要只印在图上。
- **页眉页脚**：只放文档名、版本、页码；参数与步骤写在正文主体（上下约 8% 边距会被过滤）。


| 推荐                 | 避免                |
| ------------------ | ----------------- |
| 文字步骤 + 图注 + 正文图号引用 | 纯扫描件、整页长图         |
| 图号写在步骤**同一句话**内    | 只写「见下图」、无图号       |
| 报警码/型号写在正文（可复制）   | 报警码只出现在截图或页眉页脚    |
| 书签指向页内具体段落（Y 坐标）   | 书签仅绑页码            |
| 纯文字参数表（可选中、可复制）    | 表格内嵌图片、表格做成截图/扫描图 |

## 3. 图注与图号（最重要）

系统靠**图注**和**正文图号引用**把插图挂到对应步骤。无图号时只能靠章节检索，难以在问答中按步骤展示。

### 图注（紧贴插图/表格下方，单独一行）

```
图 4-7 修改起弧时间图示
表 2-1 额定参数表
Figure 4-7 Arc start time illustration
Table 2-1 Rated parameters
```

- 行首 `图` / `表`，或 `Figure` / `Fig.` / `Table` + `章号-序号` + 说明（连字符 `-` / `–` / `—` 均可，系统归一化为 `章号-序号`）
- 不要用冒号代替空格（如 `图:4-7`、`Figure: 4-7`）
- 图注不进入正文检索，正文须单独引用图号

### 跨页拆分表格的表注

表格跨页拆成多段时，**推荐每段都加相同表号**（如各页均写 `表 2-1 额定参数表`，表号与说明一致）。系统据此识别为同一表并合并跨页片段，正文 `参数见表 2-1` 也能正确关联。

- **推荐**：每段表注含同一 `表 X-Y` 与相同说明
- **可接受**：仅在最底部一段写完整表注，其余段不写表注
- **避免**：各段表号不同、只写「续表」、或省略表号——系统可能把各段当成不同表

### 正文引用（写在步骤**同一句话**内，可跨页）

```
1. 将起弧时间设为 0.5 s（参考图 4-7）。
2. 检查扫描位置设置（参考图 4-8）。
```

常用写法：`（参考图 4-7）`、`如图 4-6 所示`、`参数见表 2-1`；英文：`see Figure 4-7`、`as shown in Fig. 4-6`、`refer to Table 2-1`。正文图号须与图注一致。

规范图号还会写入 asset 的 `figure_number`（归一化为 `4-7` 等形式），问答时 Agent 可通过 `lookup_asset` 按号精确查找插图/表格。整章操作流程、某节全部报警码等，依赖 PDF 书签标题，由 `read_section` / `lookup_toc` 定位（见 [PDF 书签](#4-pdf-书签)）。

### 跨页与位置

插图/表格挂靠正文的优先级（与 README · Asset 挂靠一致）：

1. **正文图号引用**（最可靠，**可跨任意页**）：步骤句内的 `（参考图 4-7）` 等
2. **图注反向匹配**：图注 `图 4-7 …` 与正文提及同一图号
3. **阅读顺序兜底**（仅无图号时）：同 section、最多回溯 **2 页**；同页取插图**上方**最近正文

说明与图相距超过 2 页且正文**无图号**时，很难自动关联——**必须在正文写图号**，不要依赖位置。

## 4. PDF 书签

正式手册须设**大纲/书签**，标题与正文一致；Agent 的 `lookup_toc`、`read_section` 靠书签标题匹配章节，标题含糊或口语化会导致定位失败：

- 1 级章 → 2 级节 → 3 级小节（如 `第一章` → `1.1` → `1.1.1`）
- 书签文字与用户可能提问的章节名一致（如「故障排查」而非「本章」）
- 书签目标须为**页内具体位置**（页码 + Y 坐标），不要只填「第 N 页」
- 同页多小节时，各书签落在各自标题行，不要都堆在页顶
- 在 Acrobat / Foxit 中：滚动到目标行 →「设置目标」→「缩放到」选中段落
- 「第一章」之前的前置页可能被跳过；带点线目录样式的页面会自动忽略

---

## 5. 扫描件

优先提供可选中文本层 PDF。纯扫描件须清晰可辨，页内精细切分受限，至少保证书签页级章节正确。在设置页或 `.env` 中开启 `INGEST_CAPTION_ENABLED` 时，系统可在解析期对内嵌位图做 VLM 分类描述，并尝试将栅格表 OCR 为可检索摘要，但**不能替代**正文图号引用与矢量文字表。修改手册后须**重新上传**（同名自动重索引）或请管理员触发 **重新索引**（`POST /api/v1/documents/{id}/reindex`）才会更新问答内容。

管理员可在侧边栏关闭某文档的 **参与问答**（`chat_enabled`）；关闭后即使文档状态为 `ready` 也不会进入检索范围。

---

## 6. 上传前检查

- [ ] PDF 书签完整，目标为页内具体位置（含 Y 坐标）；章节标题清晰、与正文一致
- [ ] 正式上传版已导出为 PDF；若直接上传 DOCX，已确认关键内容不依赖表格单元格或内嵌图片
- [ ] 关键信息在正文主体，非仅页眉页脚或图上
- [ ] 报警码、型号、零件号以正文文字出现（非仅截图）
- [ ] 每图单独一行图注：`图 X-Y 说明` / `表 X-Y 说明`
- [ ] 步骤句内写图号引用，一步骤一图号，与图注一致
- [ ] 参数表为文字表（非截图/扫描图），无表格内嵌图；尽量不跨页拆分，若拆分则每段加相同表号
- [ ] 已重新上传或触发重新索引

## 7. 常见问题


| 问题                | 处理                                                         |
| ----------------- | ---------------------------------------------------------- |
| 答成错误章节 / 同页小节混在一起 | 补全书签；每个书签设到对应标题行（含 Y 坐标）                                   |
| 参数检索不到            | 用可选中文本层的矢量文字表，勿用图片格式表格；关键数值在正文重复                           |
| 报警码 / 型号检索不到      | 将 `E001` 等报警码、型号写入正文（可复制文字），勿仅出现在截图或页眉页脚                    |
| 插图很少出现或位置不对       | 补图注 `图 X-Y`；步骤**同一句**内写 `（参考图 X-Y）`；图号与图注一致                |
| 问「图 X-Y 是什么」找不到     | 确认图注与正文图号一致且格式规范（如 `4-7`）；图注行须能被解析为 `figure_number`              |
| 扫描表/截图表检索弱        | 改为矢量文字表；或在设置页 / `.env` 开启 `INGEST_CAPTION_ENABLED` 辅助识别栅格表（仍建议补正文图号） |
| 跨页表只对上部分 / 表号混乱   | 尽量不拆表；必须拆分时**每段加相同表号**（如各页均写 `表 2-1 …`）；正文引用同一表号 |
| 图在下一页能对上吗         | 可以：正文句内写图号即可跨页；无图号时仅能在同 section 内回溯最多 2 页，超出则难关联 |
| 封面搜不到             | 「第一章」前内容可能被跳过，调整书签结构                                       |
| 修改后仍是旧内容          | 重新上传同名文件或触发重新索引                                            |
| 文档已 ready 但问答搜不到  | 确认 Admin 未关闭该文档的「参与问答」（`chat_enabled`）                    |


插图缺少图注时，管理员可在设置页或 `.env` 中开启 `INGEST_CAPTION_ENABLED`：解析期 VLM 会生成补充描述（`vlm_caption`），但**不能替代**正文图号引用。
