# 操作手册编写规范

本文档面向**撰写并上传至 AllDocs 的操作指南、维修手册**的编写人员，说明排版与内容组织要求，以便手册入库后获得更好的章节定位、检索命中与问答效果。

系统如何处理文档、入库配置与检索架构，见项目 [README](../README.md)。

---

## 1. 推荐格式

**正式操作手册、维修手册请优先使用带完整书签的 PDF**（保留可选中文本层，非纯扫描件）。AllDocs 的章节划分、插图/表格提取与问答展示，均围绕 PDF 版式能力设计；其他格式可用，但效果通常不如 PDF。

### 为什么推荐 PDF

| 能力 | PDF（带书签 + 文本层） | Word / Markdown 等 |
|------|------------------------|-------------------|
| **章节与目录** | 书签生成章节路径，支持按章检索、查页码 | 仅依赖标题样式，无页码与页内坐标 |
| **插图与表格** | 自动提取内嵌位图、矢量表格，识别图注 `图 X-Y` / `表 X-Y` | 图片多为独立附件，难以按页内位置关联 |
| **图号引用与展示** | 正文引用 + 空间位置挂靠，问答时按步骤句展示对应插图 | 无页内坐标，图—文关联较弱 |
| **同页多节** | 书签 Y 坐标区分页内小节边界 | 不支持 |
| **页眉页脚过滤** | 自动剔除重复页眉、页码等噪声 | 不适用 |
| **引用溯源** | 回答可定位到具体页码并在文档查看器中高亮 | 通常只能到章节，难以精确到页 |

简言之：**PDF 能同时提供「章节结构 + 页码 + 版面坐标 + 内嵌图/表」**，这正是检索、图注配对与问答插图对齐所依赖的信息；Word、Markdown 主要提供标题层级，缺少上述版式信息。

### 其他格式适用场景

| 场景 | 格式 | 说明 |
|------|------|------|
| 正式手册、需按章节检索与插图展示 | **PDF**（首选） | 设置完整书签，图注与正文引用按 [§2.6](#26-图注与正文引用) 编写 |
| 内部草稿、协作文档 | Word（.docx） | 标题使用内置「标题 1–6」样式；定稿后建议导出为 PDF 再上传 |
| 技术文档、版本管理 | Markdown（`.md`） | 用 `#` 划分章节；含大量插图时仍建议最终输出 PDF |
| 临时说明、无结构短文 | 纯文本（`.txt`） | 整篇作为一个块，不适合长篇手册 |

### PDF 编写要点（简要）

- 保留**可选中文本层**（可复制文字），不要只用扫描图代替正文
- 设置**完整书签**，目标指向页内具体段落（含 Y 坐标），见 [§3](#3-pdf-书签与章节)
- 插图下图注、正文步骤内写图号引用，见 [§2.6](#26-图注与正文引用)

**尽量避免：** 纯扫描 PDF 替代可选中文本层；整本手册压成一张长图；从 Word 导出时丢失书签或文本层。

## 2. 文字、图片与表格

可检索、可引用的主体是**文字**。图片与表格应服务于文字，而不是替代关键信息。

### 2.1 文字优先，图片作补充

- **能用文字说清楚的内容，不要用图片替代。** 操作步骤、参数数值、报警代码、注意事项、零件名称等，应写成正文或表格文字。
- **图片适合作为补充说明**：实物外观、安装位置示意、接线图、面板按键布局、软件界面截图等——文字难以精确描述、看图更直观的内容。
- 每张图附近应有**图注**（见 [§2.6 图注与正文引用](#26-图注与正文引用)），并在正文中用**统一格式**引用图号；不要只写「见下图」而无图号。
- **图号引用应写在对应操作步骤的同一句话内**（如 `将起弧时间设为 0.5 s（参考图 4-7）。`），便于系统按句关联插图，并在问答中展示在对应步骤下方。
- 有规范图注 + 正文引用的插图，入库后能与说明文字正确关联，并在问答中按步骤句展示。

### 2.2 避免重复与堆砌图片

- **同一张图不要在多处重复插入。** 多章需引用同一示意图时，正文写「参见 ×× 章图 ×-×」，只在原处保留一张图。
- 避免用大尺寸装饰图、品牌页、纯背景图填充页面——既干扰阅读，也可能无法作为有效插图被系统识别。

### 2.3 表格规范

- **表格里尽量不要嵌入图片。** 单元格内的图片难以被可靠提取，容易导致参数信息丢失。
- 参数、规格、对照信息优先用**纯文字表格**呈现；若必须图文对照，改为「正文 + 独立插图」或「文字表 + 图注」分开放置。
- 合并单元格、多层表头、旋转文字会增加解析难度；结构尽量简单、行列对齐。
- **PDF 矢量表格**（可选中文本层、非扫描图）会被整表提取为独立表格资源，并在表下识别 `表 X-Y …` 图注；正文引用格式与插图相同（见 [§2.6](#26-图注与正文引用)）。
- 扫描件中的表格无法像矢量表那样被结构化提取，关键参数仍应同时在正文中出现。

### 2.4 步骤与结构

- 操作步骤用**编号列表**（1. 2. 3.）或「步骤一 / 步骤二」书写，避免全部塞进一张流程图截图。
- 安全警告、禁止事项、扭矩值等关键信息**不要只印在图上**，须同时在正文中出现。
- 章节标题与书签（PDF）或标题样式（Word）保持一致，便于按章节定位内容。

### 2.5 快速对照

| 推荐 | 尽量避免 |
|------|----------|
| 参数表、步骤说明写成文字 | 整页扫描件替代可选中文本层 |
| 示意图 + 简短图注 | 无说明的截图或照片 |
| 纯文字表格 | 表格单元格内嵌图片 |
| 同一插图只出现一次 | 多章重复粘贴相同大图 |
| 图补充文字不好描述的内容 | 用图片代替本可打字的关键数据 |
| 书签指向具体段落（含 Y 坐标） | 书签仅绑定页码、无页内位置 |
| 关键信息写在正文主体区域 | 参数/步骤只放在页眉页脚 |
| 图注 `图 4-7 …` + 正文写 `（参考图 4-7）` | 无图号、只写「见下图」 |
| 图号写在对应步骤的**同一句话**内 | 图号与步骤分属不同句子 |
| 图与说明在同一书签章节内 | 说明与插图分属不同章且无图号引用 |

### 2.6 图注与正文引用

系统入库时会自动识别**图注**（图片/表格下方说明行）和**正文中的图号引用**，把插图挂到对应说明文字上。编写时请严格遵循下列格式，以便识别稳定、可跨页关联。

#### 图注格式（写在图片或表格正下方）

图注须为**单独一行**（不要与正文段落混在同一文本块），推荐居中放在插图正下方。

**图（插图）：**

```
图 章号-序号 简短说明
```

**表（表格截图）：**

```
表 章号-序号 简短说明
```

| 规则 | 说明 | 示例 |
|------|------|------|
| 必须以 `图` 或 `表` 开头 | 行首标识，区分正文 | `图 4-7 修改起弧时间图示` |
| 章号与序号用连字符 | 支持 `-`、`–`、`—`；号前后可有空格 | `图 4-7`、`图 4- 7` 均可 |
| 同一行仅一条图注 | 多行文字不会被识别为图注 | ❌ 图注下再跟第二行说明 |
| 紧贴插图下方 | 与图片在同一页、紧邻其下 | 不要隔页、不要夹在正文中间 |
| 一张图一条图注 | 多张并排图共用一条图注时，系统只配对最近的一张 | 尽量一图一注，或正文用图号引用 |

**推荐示例：**

- `图 4-7 修改起弧时间图示`
- `图 3-2 主电源接线端子位置`
- `表 2-1 额定参数表`

**尽量避免：**

- `图4-7`（缺少 `图` 与序号之间的空格时，部分排版下仍可能识别，但不推荐）
- `图4-7：` 使用冒号代替空格（应写 `图 4-7 说明`）
- `Figure 4-7`（英文前缀不会被识别）
- 图注写在页眉、页脚或图片上方
- 用正文段落代替图注（如长段说明紧挨图片但没有 `图 X-Y` 行首格式）

图注文字**不会**进入正文检索块，仅作为插图元数据；因此正文里仍需用下列引用格式指向该图。

#### 正文引用格式（写在说明、步骤里）

正文须用可识别的**图号引用**，系统才能把插图挂到对应段落（**可跨页**，不要求图与文字同页）。

**插图引用（任选一种，推荐前两种）：**

| 写法 | 示例 |
|------|------|
| `如图` / `参考图` / `见图` / `参见图` + 图号 | `（参考图 4-7）`、`如图 4-6 所示` |
| `图` + 图号 + `所示` | `图 4-8 所示` |
| `图` + 图号（句中提及） | `按图 4-7 设置起弧时间` |

**表格引用：**

| 写法 | 示例 |
|------|------|
| `见表` / `参考表` / `参见表` + 表号 | `（见表 3-2）` |
| `表` + 表号 + `所示` | `表 2-1 所示` |
| `表` + 表号（句中提及） | `参数见表 2-1` |

图号写法与图注一致：`章号-序号`，连字符前后可有空格（如 `4-7`、`4- 7`）。

**推荐写法（操作步骤）：**

```
处理办法：
1. 将起弧时间设为 0.5 s（参考图 4-7）。
2. 检查扫描位置设置（参考图 4-8）。
```

**尽量避免：**

- 只写「见下图」「如图所示」而**不写图号**
- 图号与图注不一致（正文写 `4-7`、图注写 `4-8`）
- 用 `[1]`、`见第 7 图` 等系统无法解析的编号方式

#### 图、文相对位置与跨页

系统按以下优先级关联插图/表格与正文：

1. **正文图号引用** → 挂到含引用的段落（**可跨页**）；入库后按**句子**建立子索引，含图号的句子优先配对对应图号资源
2. **图注识别出图号** → 反向匹配正文中提及同一图号的段落
3. **阅读顺序（空间位置）** → 同书签章节内兜底：
   - **同页**：挂到插图**正上方**、垂直位置最近的正文
   - **跨页**：插图前 **2 页**内、同章节最近的正文段落

| 场景 | 建议 |
|------|------|
| 说明在前、图在后（可隔页） | 正文中写 `（参考图 X-Y）`，且与步骤在同一句话内 |
| 图在前、说明在后 | 图注 `图 X-Y …` + 后文再次引用图号 |
| 说明与图相距超过 2 页 | **必须**在正文中写图号引用，不要依赖纯位置 |
| 同一节内多图 | 每条步骤分别写 `（参考图 X-Y）`，一图一号、一句一引 |

#### 无法关联时的表现

若图注、正文引用、阅读顺序均无法匹配，插图仍会作为**独立图块**入库（靠图注/章节检索），但不易与步骤说明在问答中**逐步展示**。**务必补全图号与图注**，不要留无说明的孤图。

#### 问答中的展示

回答中标注来源 `[n]` 后，系统会把该来源下已关联的插图/表格**插入到语义最匹配的步骤句下方**（句级对齐），而不是在文末一次性列出该来源的全部图片。

对齐依赖手册正文中的**按句图号引用**与步骤文字；编写时请保证：

- 每个操作步骤在**同一句话**内引用对应图号（如 `…（参考图 4-7）。`）
- 正文图号、图注图号、步骤语义三者一致
- 一步骤对应一图号，避免一句内混写多个无关图号

若某 chunk 内多句分别引用不同图号，问答时各图会尽量展示在各自对应的回答句下。无图号引用时，仅能通过章节/图注检索命中，较难在步骤下方精确展示。

## 3. PDF 书签与章节

正式 PDF 手册应设置**大纲 / 书签（Bookmark）**，用于章节划分与目录查询。不要用仅视觉上的大号标题代替书签。

### 3.1 层级结构

- 层级应反映文档结构，例如：
  - 1 级：章（`第一章 概述`）
  - 2 级：节（`1.1 产品简介`）
  - 3 级：小节（`1.1.1 适用范围`）
- **书签标题应与正文标题一致**，便于用户按章节名检索与理解引用来源。

### 3.2 书签目标：优先页内位置（Y 坐标），不要只绑页码

设置书签时，目标应是**某一页上的具体段落**（页码 + 垂直位置 Y 坐标），而不是「跳转到第 N 页」这种仅含页码的目标。

| 书签目标类型 | 说明 | 推荐 |
|-------------|------|------|
| **页码 + Y 坐标** | 指向页内某一行/标题（Acrobat 中「缩放到」某区域、或点击正文后创建书签） | ✅ 推荐 |
| **仅页码** | 只指定第几页，无页内高度信息 | ❌ 尽量避免 |

**为什么要用 Y 坐标：**

- 系统根据书签在页内的**垂直位置**判断该页上下各段文字属于哪个章节。
- 若多个书签在同一页且都只有页码、没有 Y 坐标，系统无法区分页内小节，整页内容会归到同一章节。
- 即使每页只有一个章节，带 Y 坐标的书签也能让目标落在标题行，章节边界更准确。

**在 PDF 编辑器中的操作要点（以 Acrobat / Foxit 为例）：**

1. 滚动到目标**小节标题或正文起始行**，再创建或移动书签，不要从书签面板里只填页码。
2. 使用「设置目标」→「缩放到」→ 选中标题或段落，确保书签落在正确行上。
3. 避免把同级书签的目标都设在**同一页的页顶**；每个书签应对应各自内容在页面上的位置。
4. 导出前抽查：点击书签，视图应跳到对应标题行，而非仅翻到页首。

### 3.3 同页多个小节

若多个书签指向同一页，**必须为每个书签设置不同的页内 Y 坐标**，落在各自小节标题或正文起始行。

- 同级小节尽量**从不同页开始**。
- 若必须在同页，各书签的垂直位置必须能区分上下内容；仅改页码、不改页内位置无效。

### 3.4 前置页与目录

- 系统可能自动跳过「第一章」之前的封面、版权等前置页。若需保留，请调整书签结构（例如将封面纳入正式章节，或核对第一章书签页码）。
- 带点线引导符的**目录页**会被自动识别并跳过入库，无需手动删除；但正文页排版不要模仿目录样式（大量点线 + 页码）。

### 3.5 页眉与页脚

入库时会**自动过滤页眉、页脚区域的重复信息**，避免文档名、章节名、页码等噪声干扰检索与问答。

| 过滤规则 | 说明 |
|---------|------|
| **上下边距区域** | 默认裁掉每页顶部、底部各约 **8%** 高度内的文本块 |
| **跨页重复文本** | 在边距区域内、且在多数页面重复出现的短文本（如手册名、章标题）会被剔除 |
| **页码样式** | 纯数字页码、`第 N 页`、`Page N` 等常见页码格式在边距内会被忽略 |

**编写建议：**

- 页眉页脚只放**装饰性或导航性**信息（文档名、版本号、页码），**不要**把参数、步骤、警告等关键内容只写在页眉页脚里。
- 若正文区域过窄、关键段落被挤进页眉页脚边距，可能被误过滤——请保证正文落在页面**中部主体区域**。
- 扫描件同样适用边距过滤；OCR 识别出的页眉页脚文字也会被剔除。

可通过环境变量调整（见 [README · 配置说明](../README.md)）：`PDF_FILTER_HEADER_FOOTER`、`PDF_HEADER_MARGIN_RATIO`、`PDF_FOOTER_MARGIN_RATIO` 等。关闭过滤可设 `PDF_FILTER_HEADER_FOOTER=false`（不推荐，除非正文确实依赖边距区域）。

## 4. Word 与 Markdown

### Word（.docx）

- 章节标题使用内置样式 **「标题 1」「标题 2」…**（Heading 1–6），不要用仅加粗的大号正文模拟标题。
- 正文、列表、表格按常规 Word 排版即可；关键数据优先落在正文或文字表中。

### Markdown（.md）

- 使用 `#` ~ `######` 划分章节。
- 纯 `.txt` 无章节结构，整篇作为一个块处理，不适合长篇手册。

### HTML

- 上传后剥除标签作为整篇正文，无章节划分，仅适合短文或临时材料。

## 5. 扫描件与图片文件

- 优先提供**带可选中文本层的 PDF**，而非纯扫描件。
- 若只能提供扫描件，请保证分辨率与对比度足够，文字清晰可辨。
- 扫描页无法按书签做页内精细切分，整页共用一个章节；仍建议补全书签，至少保证**页级**章节正确。
- 单独上传的 PNG/JPG 等图片文件仅做整图文字识别，无章节结构，不适合替代完整手册。

## 6. 上传前检查清单

**结构与章节**

- [ ] PDF 已设置完整书签层级，标题与正文一致
- [ ] 书签目标为**页内具体位置**（含 Y 坐标），非仅「第 N 页」
- [ ] 同页多个小节时，各书签落在对应段落（非全部在页顶）
- [ ] 第一章书签页码正确，避免有效正文被当作前置页跳过

**页眉页脚**

- [ ] 关键参数、步骤、警告写在**正文主体**，非仅页眉页脚
- [ ] 页眉页脚仅放文档名、版本、页码等导航信息
- [ ] 正文未挤入页面顶部/底部边距（约各 8% 区域会被过滤）

**内容与编写**

- [ ] 关键参数、步骤、警告以**正文或文字表格**呈现，非仅印在图上
- [ ] 插图有**单独一行**图注，格式为 `图 章号-序号 说明` 或 `表 章号-序号 说明`
- [ ] 操作步骤等正文内用 `（参考图 X-Y）` / `图 X-Y 所示` 等**可识别引用**（见 §2.6）
- [ ] 图号引用写在**对应步骤的同一句话**内，一步骤一图号
- [ ] 正文图号与图注图号**一致**；说明与图相距较远时必写图号，勿只写「见下图」
- [ ] 插图有图注或邻近说明；无重复粘贴的相同大图
- [ ] 表格内无嵌入图片；参数表为纯文字表
- [ ] 正文为可选中文本层（非纯扫描）或扫描质量可辨认
- [ ] 修改文档后已**重新上传**（或触发重新索引）

## 7. 常见问题

**Q：用户问某章内容，却答成别的章节？**  
A：检查书签是否完整、标题是否与正文一致。常见原因是书签**只绑了页码、没有页内 Y 坐标**，或同页多节时目标都堆在页顶；应在编辑器里把每个书签设到对应标题行。

**Q：同一页两个小节的内容混在一起？**  
A：为每个书签设置**不同的页内垂直位置**（点击目标段落后创建/更新书签），不要仅修改页码。

**Q：参数表里的数据经常检索不到？**  
A：确认表格为可编辑的矢量表而非扫描图；表格内不要嵌图片；关键数值同时在正文中写一遍。

**Q：表格里的图片看不到？**  
A：应将图片移出表格，改为正文 + 独立插图，参数写成纯文字表。

**Q：插图在回答里很少出现？**  
A：检查四项：（1）图下是否有单独一行图注 `图 X-Y …`；（2）正文步骤是否在**同一句话**内写了 `（参考图 X-Y）` 等引用；（3）图号与图注、步骤语义是否一致；（4）图与说明是否在同一书签章节。仅写「见下图」、无图号，或图号写在别的句子里，说明与图难以在问答中按步骤展示。

**Q：回答引用了 `[n]`，但插图出现在别的步骤下面？**  
A：系统按**句级语义**把图对齐到最匹配的回答句。若手册里多个图号挤在同一句、或步骤句未写图号，可能对齐不准。建议一步骤一句一图号，并在该句内写 `（参考图 X-Y）`。

**Q：图在下一页，步骤在上一页，能对上吗？**  
A：可以，但上一页正文须在与步骤**同一句话**内写 `（参考图 X-Y）` 等图号引用。若相距超过 2 页且无图号，可能无法挂到同一段说明。

**Q：多张并排截图只有一个图注？**  
A：系统按规则将图注配到最近的一张图。建议一图一注，或在正文中分别用 `（参考图 X-Y）` 引用每张图。

**Q：PDF 里的参数表能检索到吗？**  
A：可选中文本层的**矢量表格**会整表提取并在问答中展示；扫描表格或表格内嵌图片效果较差。关键参数仍应在正文或纯文字表中重复出现。

**Q：图注文字会出现在检索正文里吗？**  
A：不会。图注从正文块中剥离，仅作为插图说明；步骤文字须单独写在正文并引用图号。

**Q：封面、版权页的内容搜不到？**  
A：这些内容可能在「第一章」之前被跳过；若需检索，调整书签或章节结构。

**Q：页眉里的文档名/页码会出现在问答里吗？**  
A：不会。系统入库时会过滤页眉页脚边距内的重复文本与页码。请勿把关键操作说明只写在页眉页脚。

**Q：修改手册后问答还是旧内容？**  
A：必须重新上传文档或触发重新索引，系统不会自动同步本地文件变更。
